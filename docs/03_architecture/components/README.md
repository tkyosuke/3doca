# components - コンポーネント図（Level 3 / Tier 4）

## 目的

内部構造の詳細を示す。クラス・モジュール関係と実装詳細へのリンクを提供。

## ティア仕様

| 項目 | 値 |
|------|-----|
| **C4レベル** | Level 3 |
| **ティア** | Tier 4 |
| **ビジュアル/テキスト比率** | 25/75 |
| **対象読者** | 開発者、メンテナー |
| **更新頻度** | 高（変更が多いため） |

## コンポーネント図の要件

1. **内部構造の詳細を示す**
2. **クラス・モジュール関係を図示**
3. **実装詳細へのリンクを含む**
4. **変更が多いため、メンテナンスコストを考慮**

## 注意事項

> ほとんどのチームは、コンテキスト+コンテナ図（C4レベル1-2）が十分なアーキテクチャドキュメントを提供し、コンポーネントとコード図は変更が多いためメンテナンスする価値がないことが多いと気づきます。
> — C4モデル
