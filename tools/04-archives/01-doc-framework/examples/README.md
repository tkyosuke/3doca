## 実例詳細

### データ解析プロセス (`00-data-analysis-process.md`)

**テンプレート**: `process-document-template.md`

**内容**:
- Pandas/NumPyを使用した標準的なデータ解析ワークフロー
- データ取得→前処理→分析→報告の4フェーズプロセス
- 各フェーズの具体的な手順と品質チェックポイント
- Mermaidフローチャートによる全体像の可視化

**学習ポイント**:
- 一般的なデータ解析プロセスの標準構造
- フェーズ別の責任分界とアウトプット定義
- 品質ゲートの適切な配置

**対象読者**: データアナリスト（Python中級レベル）

- Mermaidフローチャートによる可視化

**対象読者**: データ解析初心者〜中級者

### データ品質問題対応プレイブック (`data-quality-issues-playbook.md`)

**テンプレート**: `playbook-template.md`

**内容**:
- データ品質問題（欠損値・異常値・不整合）の診断と対処フレームワーク
- 影響度評価（Critical/High/Low）に基づく対応優先度判定
- Mermaid意思決定ツリー（緊急対応/計画対応/標準対応の分岐）
- 根本原因分析とエスカレーション基準

**学習ポイント**:
- データ品質問題の体系的な分類と対処方法
- 影響度評価の定量的基準（欠損率、異常値比率）
- トラブルシューティング寄りのプレイブック構造

**対象読者**: データエンジニア、データアナリスト

**異常検知判断プレイブックとの違い**: 品質問題は「データそのものの欠陥」に対処、異常検知は「統計的な外れ値の解釈」を支援

## ドメイン別応用例

### CFD（数値流体力学）ドメインへの応用

1. **データ品質分析プロセス** → **メッシュ品質分析プロセス**
   - データプロファイリング → メッシュ品質チェック（非直交性、スキューネス）
   - データ検証 → 境界条件の物理的整合性確認

2. **異常検知判断プレイブック** → **収束判定プレイブック**
   - 統計的有意性 → 残差の収束判定
   - ビジネス影響度 → 物理量の妥当性評価

3. **データクレンジングランブック** → **メッシュ修正ランブック**
   - 欠損値処理 → セル品質改善
   - 外れ値処理 → 境界レイヤー調整

### GIS（地理情報システム）ドメインへの応用

1. **データ品質分析プロセス** → **空間データ品質分析プロセス**
   - データプロファイリング → ジオメトリ検証、投影系確認
   - データ検証 → トポロジーチェック

2. **異常検知判断プレイブック** → **空間異常パターン判断プレイブック**
   - 統計的有意性 → 空間自己相関の評価
   - ビジネス影響度 → 地域特性の考慮

## RAG統合への準備

全ての実例は将来的なRAG統合を見据えて設計されています：

### フロントマターの活用

```yaml
# RAGフィルタリング例
category: process         # プロセスドキュメントのみ検索
domain: data-analysis     # データ解析ドメインのみ検索
difficulty: intermediate  # 中級者向けのみ検索
```

### セマンティック検索最適化

- **1文書1概念**: 各実例は単一の主題（データ品質分析、異常検知等）
- **自己完結型セクション**: 各セクションが独立して検索可能
- **代名詞の回避**: 具体的な用語を繰り返す（「データ品質分析では...」）
- **チャンクサイズ最適化**: セクションは250〜512トークン目安

### PostgreSQL/pgvector統合例

```sql
-- データ解析関連ドキュメントのセマンティック検索
SELECT title, content, 1 - (embedding <=> query_embedding) AS similarity
FROM documents
WHERE metadata->>'domain' = 'data-analysis'
  AND metadata->>'category' = 'process'
ORDER BY embedding <=> query_embedding
LIMIT 5;
```
